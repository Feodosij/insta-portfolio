# Портфоліо флориста — дизайн MVP

## Контекст

Сайт-портфоліо флориста. Основний потік трафіку — з Instagram (посилання в біо/сторіз), тож досвід має бути mobile-first. Мета цієї ітерації — MVP, який можна показати SMM-менеджеру і далі доопрацьовувати.

Мова інтерфейсу та контенту сайту: **українська**.

## Архітектура і стек

Одне Next.js (App Router, TypeScript) застосування, деплой на **Vercel** (free tier). Публічний сайт і адмінка — один проєкт, різні роути:

- `/` — публічний сайт
- `/admin` — захищена панель керування контентом

| Призначення | Сервіс | Навіщо |
|---|---|---|
| Дані (таби, порядок фото, тексти) | Supabase Postgres (free) | Легка реляційна БД, вкладається у free tier |
| Автентифікація адмінки | Supabase Auth | Один користувач, email+пароль |
| Зберігання і оптимізація фото | Cloudinary (free, до 25GB) | Автооптимізація, різні розміри під мобільні екрани, upload з телефону |
| Стилі | Tailwind CSS | — |
| Домен | піддомен Vercel (`*.vercel.app`) | Достатньо для MVP, свій домен — пізніше |
| Аналітика | Vercel Analytics (free) | Базова статистика переглядів для звіту SMM |

Очікуваний обсяг: 50–300 фото — вкладається у безкоштовні ліміти всіх сервісів.

## Публічна частина

### Головна (`/`)
- Повноекранне фонове фото
- Ім'я/назва по центру
- Кнопка **"Переглянути"**
- Клік по кнопці або скрол вниз → плавний перехід до галереї (той самий URL, без переходу на іншу сторінку)
- Технічний нюанс: висота блоку рахується через `100dvh` (не `100vh`) — інакше в вбудованому браузері Instagram адресний рядок з'їдає частину екрана і фото/кнопка "стрибають"

### Галерея
- Masonry-розкладка (картки різної висоти), референс — Pixover/Pinterest-стиль
- Дефолтний таб **"Всі"** показується першим — усі фото без фільтра, поки користувач не обрав конкретну категорію
- Таби зверху для фільтрації за категоріями (наприклад: Конструкції, Весілля тощо) — **набір табів повністю керується з адмінки**, не зашитий у код
- Клік по фото → повноекранний перегляд зі свайпом між фото
- Ліниве довантаження (infinite scroll) — при 100-300 фото не можна рендерити все одразу, фото підвантажуються порціями під час скролу

### Розділ "Послуги / Як замовити"
- Короткий текстовий блок (редагується з адмінки)
- Кнопки-посилання: **Instagram DM** та **Telegram**
- Без форми замовлення на сайті — MVP навмисно спрощений, клієнт пише напряму в месенджер

## Адмінка (`/admin`)

Доступ: один користувач, вхід через email+пароль (Supabase Auth). Інтерфейс спроєктований під керування з браузера телефону — прості списки й форми, без складних редакторів.

### Керування фото
- Список фото галереї з превʼю
- Завантаження нових (вибір із галереї телефону або камери). Стиснення в WebP робиться вручну перед завантаженням (наприклад через Squoosh) — адмінка окрему автоматичну компресію на MVP не робить, орієнтується на вже оптимізовані файли
- Зміна порядку — drag-and-drop з підтримкою touch
- Привʼязка фото до одного чи кількох табів
- Видалення фото — з підтвердженням (модалка "Точно видалити?"), щоб не змахнути випадково на телефоні

### Порожні стани
- Таб без жодного фото — показується заглушка замість порожньої сітки
- Перший запуск (взагалі без фото) — заглушка на публічному сайті замість пустої галереї

### Керування табами
- Список табів з порядком
- Додати / перейменувати / видалити / переставити таб

### Керування текстом
- Ім'я на головній
- Текст блоку "Послуги / Як замовити"
- Посилання на Instagram і Telegram
- Заміна фонового фото головної сторінки

## Технічні деталі сторінки

- Favicon, заголовок вкладки браузера
- Open Graph теги (title, description, превʼю-зображення) — на випадок, якщо посилання на сайт поширюватимуть поза Instagram (месенджери, сторіз з посиланням)

## Модель даних (Supabase)

- **categories** — `id`, `name`, `order`
- **photos** — `id`, `cloudinary_url`, `order`, `category_ids` (звʼязок many-to-many з categories)
- **site_content** — key/value: ім'я на головній, hero-фото, текст розділу послуг, посилання на месенджери

## Поза межами MVP (свідомо не робимо зараз)

- Форма замовлення на сайті (тільки посилання на месенджери)
- Багатомовність
- Декілька адмін-користувачів / ролі
- Свій домен (можна підключити пізніше без переробки)
- Автоматичне стиснення/конвертація фото в адмінці (робиться вручну через Squoosh перед завантаженням)
- Блок "Про флориста" — не запитувалось, додати легко пізніше за потреби

## Відкриті питання / на уточнення пізніше

- Точний текст і фото для головної сторінки (hero) — надасть флорист/SMM
- Початковий список табів-категорій (Конструкції, Весілля, ...) — фінальний перелік
